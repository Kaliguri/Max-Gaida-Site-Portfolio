# Структура резюме

Два резюме делят данные (`content/`), но по-разному расставляют акценты. Один человек — две входные точки.

## Общий каркас

1. **Hero**: роль + одна фраза-позиционирование + фото + кнопки (Telegram, скачать PDF).
2. **«Я также работаю как X»**: 2-3 строки + явный переход на второе резюме.
3. **Summary** — короткая сводка.
4. **Skills** — сгруппированные навыки.
5. **Опыт / Проекты** — с «my role» и GitHub-ссылками (для технических). Выдвижные детали, геймплей-видео.
6. **Образование / релевантное**.
7. **Контакты**: Telegram (явно), email (`mailto`-fallback), GitHub.

## Акценты по ролям

- **Game Designer**: механики, системный дизайн, UX, баланс, документация, плейтесты.
  Проекты — через призму дизайн-решений.
- **Unity Developer**: C# / Unity, архитектура, инструменты редактора, перф, тесты.
  Проекты — через призму кода и архитектуры.

## Кросс-связь

Каждое резюме в блоке «я также работаю как X» коротко продаёт вторую роль и ведёт на её резюме.
