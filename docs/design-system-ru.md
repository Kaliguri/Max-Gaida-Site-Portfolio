# Дизайн-система — Max Gaida portfolio

Справочник по визуальным токенам и правилам их применения. Источник токенов —
[`src/app/globals.css`](../src/app/globals.css); здесь описано, **как** ими пользоваться.
Design-system reference: tokens live in `globals.css`, this doc is the usage contract.

Если код расходится с этим документом — прав документ, код правим.

---

## 1. Токены цвета

Одна палитра, два режима (`:root`/`.dark` — тёмный по умолчанию, `.light` — тёплая
«бумага»). Компоненты **никогда** не используют hex напрямую — только семантические
Tailwind-утилиты, которые мапятся на CSS-переменные (см. `@theme inline`).

| Токен          | Утилиты                                     | Роль                                               |
| -------------- | ------------------------------------------- | -------------------------------------------------- |
| `background`   | `bg-background`                             | Фон страницы.                                      |
| `surface`      | `bg-surface`                                | Карточки, приподнятые поверхности.                 |
| `surface-2`    | `bg-surface-2`                              | Hover/inset внутри поверхностей.                   |
| `foreground`   | `text-foreground`                           | **Основной** текст и заголовки.                    |
| `muted`        | `text-muted`                                | **Вторичный** текст: описания, подписи, метки.     |
| `border`       | `border-border`, `bg-border`                | Хайрлайны, разделители, неактивные индикаторы.     |
| `accent`       | `text-accent`, `bg-accent`, `border-accent` | Янтарь: эйбрау, ссылки, hover, активные состояния. |
| `accent-hover` | `text-accent-hover`                         | Наведение на янтарные элементы.                    |

Правило: цвет задаём токеном. Появилась потребность в новом оттенке — сначала токен в
`globals.css`, потом использование. Прямой hex в компоненте — ошибка ревью.

## 2. Иерархия текста (главное правило)

Три уровня, и они **не** инвертируются от секции к секции:

1. **Основная информация → `text-foreground`** (белый/тёмный).
   Заголовки секций (`h2`), названия сущностей, значения фактов, важные инлайн-акценты.
2. **Вспомогательная информация → `text-muted`** (серый/тауп).
   Описания, лиды, подписи-метки, второстепенные строки.
3. **Эйбрау и интерактив → `text-accent`** (янтарь).
   Надзаголовки-«ярлычки» над блоком, ссылки, «смотреть →», активные пункты навигации.

### Почему `<dl>` иногда «серый сверху», иногда «белый сверху»

Это **не** несогласованность, а то же правило. Смотрим, что здесь основная информация:

- **Список «имя → описание»** (Soft Skills, навыки образования): основное — имя →
  `dt` белый (`text-foreground`), `dd` серый (`text-muted`).
- **Метаданные «метка → значение»** (программа обучения: «Вуз», «Факультет»): основное —
  значение → `dt`-метка серая (`text-muted`), `dd`-значение белое (`text-foreground`).

В обоих случаях белым выделена суть, серым — подпись. Инверсии нет.

## 3. Типографика

- Шрифт — Onest (`--font-sans`), через `font-family` на `body`.
- **Заголовок секции** (`h2`): `text-2xl sm:text-3xl font-semibold tracking-tight`, `text-foreground`.
- **Эйбрау подсекции** (`h3`): `text-sm font-medium tracking-wide uppercase`, `text-accent`.
  Это единый токен надзаголовка — тот же в hero, в секциях и в карусели
  (`tracking-wide`, не `tracking-widest`; `text-sm`, не `text-xs`).
- **Тело**: базовый размер, `leading-relaxed`, `text-muted`.
- **Заголовок H1 (hero)**: `text-4xl sm:text-6xl font-semibold tracking-tight leading-[1.05]`.

---

## 4. Ритм секций

- Контентная колонка: `mx-auto w-full max-w-5xl px-6 sm:px-10`.
- Разделение секций: верхний хайрлайн `border-border border-t` + `pt-8 sm:pt-10`.
- Якорь под липкую шапку: секции `scroll-mt-20`, подсекции `scroll-mt-24`.
- Подсекции внутри секции: `mt-10` (первая) / `mt-12` (последующие).

## 5. Компоненты-паттерны

- **Интерактивная карточка** (проект, резюме, витрина): `border-border bg-surface
rounded-xl border`, `hover:border-accent transition-colors`. Кликается целиком — одна
  большая кнопка/ссылка. Реакция на hover обязательна (акцентный контур на **всей** карточке).
- **Статичная инфо-карточка** (грани «Основное», soft skills, навыки образования):
  `border-border bg-surface/40 rounded-xl border p-5/p-6`. **Без** hover — она ничего не
  открывает, реакция бы только отвлекала. Единый контейнер для карточных сеток контента.
- **Круглая контрол-кнопка** (стрелки карусели, «наверх»): `border-border bg-surface
rounded-full border`, `hover:border-accent hover:text-accent`.
- **Пилюля-контакт** (шапка/футер): `border-border rounded-full border text-muted`,
  `hover:border-accent hover:text-foreground`.
- **Индикатор-точка / линия** (пейджер, таймер, TOC-риска): активный `bg-accent`,
  неактивный `bg-border`.
- **Фокус**: если весь блок — одна большая кнопка, гасим нативный контур на кнопке
  (`focus:outline-none`) и рисуем акцентный контур на всей карточке через
  `focus-within:border-accent` (тот же контур, что и на hover), а не на вложенном элементе.

---

## 6. Движение

- Ревилы/hover/слайды — CSS/Framer Motion; тяжёлый скролл-скраб hero — GSAP ScrollTrigger.
- **Всегда** уважаем `prefers-reduced-motion: reduce` — анимации выключаются или заменяются
  мгновенным состоянием, контент никогда не остаётся скрытым.
- Для анимации прогресса/заполнения предпочитаем `transform` (composited), не `width`.

## 7. Чек-лист ревью

- [ ] Цвета — только токены, без прямого hex.
- [ ] Основная инфа `foreground`, вспомогательная `muted`, эйбрау/интерактив `accent`.
- [ ] Эйбрау-`h3` — единый токен (`text-sm tracking-wide uppercase text-accent`).
- [ ] Карточки/кнопки/пилюли следуют паттернам из раздела 5.
- [ ] Есть ветка `prefers-reduced-motion`.
- [ ] Проверено в обеих темах (dark/light).
